# ChurchAfrica ChMS - Comprehensive Branding Guidelines

## Brand Overview

**Brand Name:** ChurchAfrica ChMS  
**Tagline:** "Africa-First Church Management Made Simple"  
**Mission:** Empowering African churches with modern, offline-first technology  
**Vision:** The leading church management solution across Africa  

## Brand Personality

### Core Values
- **Simplicity**: Easy to use, intuitive interface
- **Reliability**: Works offline, always available
- **Community**: Built for African churches, by African developers
- **Innovation**: Modern technology with traditional values
- **Accessibility**: Inclusive design for all users

### Brand Voice
- **Warm & Welcoming**: Friendly, approachable tone
- **Professional**: Trustworthy and competent
- **Empowering**: Helps churches grow and thrive
- **Respectful**: Honors church traditions and culture
- **Supportive**: Always ready to help and guide

## Visual Identity

### Primary Logo
**Logo Concept:** Modern cross integrated with African geometric patterns
- **Primary Version**: Full logo with text "ChurchAfrica ChMS"
- **Icon Version**: Symbol only for app icons and favicons
- **Horizontal Version**: For headers and wide spaces
- **Stacked Version**: For narrow spaces and mobile

### Color Palette

#### Primary Colors
1. **Garnet Night** - `#2D1B69` (Primary Brand Color)
   - Usage: Headers, primary buttons, app theme
   - Represents: Spirituality, depth, trust

2. **Golden Dawn** - `#FFB800` (Accent Color)
   - Usage: Call-to-action buttons, highlights, success states
   - Represents: Hope, prosperity, African sunshine

3. **Pure White** - `#FFFFFF` (Light Background)
   - Usage: Light theme backgrounds, text on dark
   - Represents: Purity, clarity, simplicity

#### Secondary Colors
4. **Deep Charcoal** - `#1A1A1A` (Dark Text)
   - Usage: Primary text, dark theme elements
   - Represents: Strength, stability

5. **Soft Gray** - `#F5F5F5` (Light Gray)
   - Usage: Subtle backgrounds, borders, disabled states
   - Represents: Balance, neutrality

6. **Success Green** - `#10B981` (Success States)
   - Usage: Success messages, positive indicators
   - Represents: Growth, success, life

7. **Warning Amber** - `#F59E0B` (Warning States)
   - Usage: Warning messages, caution indicators
   - Represents: Attention, caution

8. **Error Red** - `#EF4444` (Error States)
   - Usage: Error messages, critical alerts
   - Represents: Urgency, attention needed

## Implementation Guidelines

### Logo Usage

#### Do's
‚úÖ Use official logo files only  
‚úÖ Maintain minimum clear space (logo height x 0.5)  
‚úÖ Use appropriate version for context  
‚úÖ Ensure sufficient contrast with background  
‚úÖ Scale proportionally  

#### Don'ts
‚ùå Don't stretch or distort the logo  
‚ùå Don't change colors without approval  
‚ùå Don't add effects (shadows, outlines, etc.)  
‚ùå Don't place on busy backgrounds  
‚ùå Don't use low-resolution versions  

### Color Usage

#### Accessibility Requirements
- **Contrast Ratio**: Minimum 4.5:1 for normal text
- **Large Text**: Minimum 3:1 for 18pt+ or 14pt+ bold
- **Interactive Elements**: Minimum 3:1 contrast ratio
- **Color Blindness**: Don't rely on color alone for meaning

#### Color Combinations
‚úÖ **Approved Combinations**:
- Garnet Night background + White text
- White background + Deep Charcoal text
- Golden Dawn background + White text
- Soft Gray background + Deep Charcoal text

‚ùå **Avoid Combinations**:
- Golden Dawn background + Garnet Night text (poor contrast)
- Soft Gray background + White text (insufficient contrast)
- Error Red background + Warning Amber text (confusing)

### Typography Guidelines

#### Hierarchy
1. **H1 (Page Title)**: Poppins SemiBold 32px
2. **H2 (Section Title)**: Poppins Medium 24px
3. **H3 (Subsection)**: Poppins Medium 20px
4. **H4 (Component Title)**: Inter SemiBold 18px
5. **Body Text**: Inter Regular 16px
6. **Small Text**: Inter Regular 14px
7. **Caption**: Inter Regular 12px

#### Line Height
- **Headings**: 1.2x font size
- **Body Text**: 1.5x font size
- **Small Text**: 1.4x font size

#### Letter Spacing
- **Headings**: -0.02em (tighter)
- **Body Text**: 0em (normal)
- **All Caps**: 0.05em (looser)

## Brand Asset Creation Guide

### Logo Creation Process
1. **Concept Development**
   - Sketch initial concepts combining cross and African patterns
   - Test readability at various sizes (16px to 512px)
   - Ensure cultural sensitivity and appropriateness

2. **Digital Creation**
   - Create in vector format (Adobe Illustrator or Figma)
   - Use brand colors exactly as specified
   - Maintain consistent stroke weights and proportions

3. **Optimization**
   - Export SVG for web use
   - Create PNG versions for various sizes
   - Optimize file sizes without quality loss

### Icon Creation Workflow
```bash
# Icon generation script
# Create all required PWA icon sizes from master 1024x1024 source

# Standard PWA icons
convert icon-master.png -resize 192x192 icon-192x192.png
convert icon-master.png -resize 512x512 icon-512x512.png

# iOS Safari icons
convert icon-master.png -resize 180x180 apple-touch-icon.png

# Favicons
convert icon-master.png -resize 32x32 favicon-32x32.png
convert icon-master.png -resize 16x16 favicon-16x16.png

# Maskable icon (with safe area)
convert icon-master-maskable.png -resize 192x192 icon-192x192-maskable.png
convert icon-master-maskable.png -resize 512x512 icon-512x512-maskable.png
```

### Figma AI Compatibility Notes

#### Auto Layout Requirements
- **Use Auto Layout**: All components must use Figma's auto layout for consistency
- **Spacing Variables**: Reference spacing tokens in auto layout padding/margins
- **Responsive Design**: Use auto layout for responsive component behavior
- **Component Nesting**: Maintain proper component hierarchy with auto layout

#### Component Reuse Guidelines
- **Master Components**: Create master components for buttons, cards, inputs
- **Instance Overrides**: Use instance overrides for text and color variations
- **Nested Components**: Build complex components from smaller reusable parts
- **State Variants**: Create component variants for different states (hover, active, disabled)

#### Variable Naming for AI
- **Color Variables**: Use semantic names (primary, accent, success, warning, error)
- **Typography Variables**: Reference font families and weights consistently
- **Spacing Variables**: Use consistent spacing scale references
- **Component Variables**: Name components with clear hierarchy (btn/primary, card/event)
